UX/UI Design

Royal LePage Usability Test

Project Overview

This project is to conduct usability testing on the Royal Lepage website using Lean UX methodology and identify the changes required to improve user performance and satisfaction as a team.

Timeline Nov 2020 - Dec 2020
Role UX UI Designer, UX Researcher
Tools Adobe XD, Google Forms
01

Understand

In this stage, we first analyzed the main audience of their website, and build a user profile and scenario. After that, we identified some business problems and built a list of assumptions based on those problems. Lastly, to test those assumptions, we developed hypotheses to test our assumptions.

02

Test

Our usability testing consisted of three parts inclduing screeners, user tasks and SUS questions. Through screeners, we narrowed the participants down to three who are most likely to use the website in order to proceed to user tasks. Due to current situations, all the testings were conducted remotely on Zoom. We screenrecorded a session with participants' permission and asked them to think out loud while doing the tasks to better understand their thought processes.

03

Redesign

Based on the testing results, we were able to identify three main pain points, which are poor access to the important information, limited advanced search capabiliteis and unnecessary information.

In order to improve overall user experience, we recommended to add more filters to the advanced search capabilities to facilitate easier searching ,and make the search result page more straight forward by displaying important property information on each card and make the images scrollable to keep users engaged.
